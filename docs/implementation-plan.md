# 家系図SVG CLI 実装計画（Graphviz廃止版）

## Summary

- 現在の `DotBuilder -> dot -> SVG` を廃止し、Ruby内でレイアウト計算とSVG描画を完結させる。
- 対象は `samples/family-showcase.ftree` を読みやすく描くこと（複数世代、兄弟、再婚、片親）。
- CLI外部仕様は維持し、破壊的変更は内部実装と依存関係（Graphviz削除）に限定する。
- 入力は簡易記法を主軸に、GEDCOMサポートは継続する。

## Implementation Changes

- 新レイアウトエンジンを導入し、内部モデルを「人物ノード」「婚姻ユニット」「兄弟グループ」で扱う。
- 世代レイヤーを親子制約の最長距離で決定し、配偶者を同一レイヤーに固定する。
- 同レイヤー内の並びは `birth_year` と `id` で初期化し、barycenter法の上下スイープで交差を削減する。
- 制約として「夫婦隣接」「兄弟連続」をブロック化して順序崩れを防ぐ。
- 座標は固定メトリクス（ノード幅/高さ、世代間隔、兄弟間隔、家族間隔）で決定し、出力を決定論にする。
- エッジは直交ルーティングで、`婚姻ユニット -> 兄弟バー -> 子ノード` を標準化する。
- レンダラーをDOT前提からレイアウト済み描画モデル前提に置換し、SVGを直接出力する。
- Graphviz依存コード・要件を削除し、READMEを更新する。

## Public API / Output Policy

- 維持:
  - `family-tree render INPUT -o OUTPUT.svg [--format auto|gedcom|simple] [--strict]`
- 変更:
  - Graphviz（`dot`）を不要化し、`dot` 未導入エラーを廃止する。
- SVG:
  - 静的SVGを安定出力（同一入力で同一構造・同一座標）。
  - レイヤー順は `edge -> node -> label` に固定。

## Test Plan

- レイアウト単体:
  - 夫婦が同世代で隣接する。
  - 兄弟が同世代で連続する。
  - 親世代と子世代が分離される。
  - 再婚時に人物重複ノードが発生しない。
- 描画単体:
  - SVG必須要素（ノード、ラベル、接続線）を含む。
  - 無効な path を出力しない。
- 統合:
  - `samples/family-showcase.ftree` で主要交差が少ない読みやすい図を生成する。
  - GEDCOM入力でも同一エンジンでSVG生成できる。
- 回帰:
  - CLIの既存エラー系（入力不足、`--strict`、未知フォーマット）を維持する。

## Assumptions

- MVP完了基準は「夫婦隣接・兄弟整列・世代分離・主要交差なし」。
- 交差ゼロ最適化はMVP対象外とし、必要なら次フェーズで追加する。
- 片親ケースは単親モード婚姻ユニットで統一的に処理する。
